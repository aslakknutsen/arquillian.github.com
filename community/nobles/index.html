Arquillian Nobles

This page recognizes community members that have helped shape Arquillian during each release cycle, either though code contributions, participation or advocacy. For donating their time, effort and patience to make Arquillian a better testing framework for the benefit of the community, we award each of them the title Arquillian Noble.

Community members may be crowned multiple times. Once a contributor becomes a team member, subsequent contributions are not listed here (team members are noble by default).

Advocates, Innovators and Early Adopters

  • Pete Muir – Co-founder of Arquillian. Created JBoss Test Harness, the precursor to Arquillian.
  • Cheyenne Weaver – Brought Ike to life and gave Arquillian a visual identity
  • Adrian Cole – Took Arquillian to the clouds (via jclouds); advocates it as a Java cloud power tool
  • Paul Bakker – Created and maintains the Arquillian Forge plugin (SEAMFORGE-146)
  • Jesper Pedersen – Implemented the IronJacamar (embedded JCA) container adapter
  • German Escobar – Wrote about testing a JMX CDI extension with Arquillian
  • Michael Schuetz – Setup the Arquillian sandbox repository, created quickstart projects, researched and wrote Seam 2 extension
  • Adam Warski – Worked out a solution for doing efficient database-oriented testing
  • Markus Eisele – Covered Arquillian in Java Tech Journal March 2011
  • Cosmin Martinconi (mentor: Jakob Korherr) – GSoC 2010 project to create an automated webapp test suite for MyFaces based on the Arquillian SPI (wrap-up)
  • John Ament – Wrote about testing a property loader CDI extension with Arquillian
  • Stuart Douglas – Converted Weld extensions tests to Arquillian, uses Arquillian to test fakereplace
  • Jordan Ganoff – Wrote about testing the Seam JMS module with Arquillian
  • Lincoln Baxter III – Wrote about CDI-powered unit testing with Arquillian, uses Arquillian to test Seam Forge
  • Mike Brock – Adopted Arquillian for testing Errai
  • Bob McWhirter and Jim Crossley – Experimented combining Arquillian with JRuby and RSpec in TorqueBox

Thanks to all of you who have tweeted about #Arquillian as well!
